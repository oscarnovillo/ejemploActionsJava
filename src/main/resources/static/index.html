Sistema de Gestión Hospitalaria NACHETE

Una aplicación REST para la gestión de pacientes y registros médicos

Descripción del Proyecto

Este proyecto implementa un sistema de gestión hospitalaria completo utilizando Spring Boot como backend. La aplicación permite administrar pacientes, doctores, registros médicos y medicamentos prescritos, proporcionando una API REST completa que facilita la integración con aplicaciones cliente.

Gestión de Pacientes

Permite crear, consultar, actualizar y eliminar información de pacientes, incluyendo datos personales y credenciales de acceso.

Registros Médicos

Administra registros médicos asociados a pacientes, con diagnósticos, fechas de admisión y medicamentos prescritos.

Sistema de Pagos

Seguimiento de pagos asociados a cada paciente, facilitando la gestión financiera del hospital.

Autenticación

Sistema de autenticación de usuarios mediante credenciales almacenadas de forma segura.

Stack Tecnológico

  • Java 21
  • Spring Boot 3.2
  • Spring JDBC
  • MySQL Database
  • Lombok
  • Docker
  • Maven
  • GitHub Actions para CI/CD
  • TestContainers para pruebas

Endpoints API REST

La API expone los siguientes endpoints principales:

  • GET /patients - Obtener todos los pacientes
  • GET /patients/{id} - Obtener un paciente específico
  • POST /patients - Crear un nuevo paciente
  • PUT /patients - Actualizar un paciente existente
  • DELETE /patients/{id} - Eliminar un paciente
  • GET /patients/{idPatient}/medRecords - Obtener registros médicos de un paciente
  • POST /patients/medRecords - Crear un nuevo registro médico
  • PUT /patients/medRecords - Actualizar un registro médico
  • DELETE /patients/medRecords/{id} - Eliminar un registro médico
  • GET /doctors - Obtener todos los doctores
  • POST /login - Autenticar usuario

Arquitectura

El proyecto sigue una arquitectura en capas que separa claramente las responsabilidades:

  • Capa UI: Controladores REST que manejan las peticiones HTTP
  • Capa de Servicio: Lógica de negocio y transformación de datos
  • Capa de Acceso a Datos: Repositorios y mappers para interactuar con la base de datos
  • Modelo de Dominio: Entidades y objetos de transferencia de datos
